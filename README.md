
# 🛒 سامانه خرید و فروش کالای دست دوم (Second-Hand Marketplace)

> **پروژه پایانی درس برنامه‌سازی پیشرفته**

سامانه خرید و فروش کالای دست دوم یک نرم‌افزار مبتنی بر معماری **Client-Server** است که با هدف شبیه‌سازی یک سامانه واقعی خرید و فروش آنلاین طراحی و پیاده‌سازی شده است. در این پروژه، **Backend** با استفاده از **Spring Boot** و **Frontend** با استفاده از **JavaFX** توسعه یافته‌اند و ارتباط بین آن‌ها از طریق **REST API** برقرار می‌شود.

در طراحی پروژه تلاش شده است اصول **برنامه‌نویسی شیءگرا (OOP)**، **معماری لایه‌ای (Layered Architecture)**، **مدیریت امنیت**، **اعتبارسنجی داده‌ها** و **ذخیره‌سازی پایدار اطلاعات** رعایت شود.

# 🏗 معماری پروژه

پروژه بر اساس معماری **Client-Server** و الگوی **Layered Architecture** طراحی شده است.

```
JavaFX Client
        │
 REST API
        │
Spring Boot Backend
        │
Service Layer
        │
Repository Layer
        │
PostgreSQL Database
```

در این ساختار تمامی منطق برنامه در سمت Backend قرار دارد و Frontend تنها وظیفه تعامل با کاربر و ارسال درخواست‌ها به سرور را بر عهده دارد.

# 🚀 نحوه اجرای Backend

### پیش‌نیازها

* Java JDK
* Maven
* PostgreSQL

ابتدا یک پایگاه داده PostgreSQL ایجاد کرده و اطلاعات اتصال را در فایل زیر تنظیم کنید:

```
backend/src/main/resources/application.properties
```

سپس پروژه را اجرا کنید:

```bash
cd backend
mvn spring-boot:run
```

هنگام اجرای پروژه، تمامی Migrationهای پایگاه داده توسط **Flyway** به صورت خودکار اعمال می‌شوند.

---

# 💻 نحوه اجرای Frontend

```bash
cd frontend
mvn javafx:run
```

> برای اجرای صحیح Frontend، ابتدا Backend باید در حال اجرا باشد.

# 📈 وضعیت توسعه پروژه

در طول توسعه پروژه، تمرکز اصلی بر طراحی و پیاده‌سازی منطق سمت سرور و زیرساخت سامانه بوده است؛ به همین دلیل بخش Backend از نظر قابلیت‌ها کامل‌تر از رابط کاربری توسعه یافته است.

در نسخه فعلی:
 

| Backend  | بخش عمده قابلیت‌های اصلی سامانه پیاده‌سازی شده است.                                                                                                       
| Frontend | قابلیت‌های اصلی رابط کاربری پیاده‌سازی شده و توسعه سایر بخش‌ها در ادامه قابل انجام است.                                                               
| Database | ساختار اصلی پایگاه داده طراحی و پیاده‌سازی شده است. برخی قسمت‌ها مانند مدیریت داده‌های City و Attribute قابلیت بهبود و بازطراحی در نسخه‌های بعدی را دارند.
